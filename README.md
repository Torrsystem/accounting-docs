# เช็คลิสต์ + รวมเอกสารส่งบัญชี

เว็บแอปหน้าเดียว (ไม่มี backend ของตัวเอง) สำหรับทำเช็คลิสต์เอกสารที่ต้องส่งบัญชีทุกเดือน แนบไฟล์จริงต่อรายการ เก็บถาวรผ่าน Supabase Storage แล้วรวมเป็น PDF ไฟล์เดียวเพื่อดาวน์โหลด/ปริ้นท์

## โครงสร้าง

- ไฟล์เดียว: `index.html` — เป็นทั้งหน้าตาและโค้ดทั้งหมด เปิดจากเครื่องหรือโฮสต์บนเว็บก็ได้
- ไม่มีเซิร์ฟเวอร์ของตัวเอง ข้อมูลเอกสารจริงเก็บอยู่ที่ **Supabase Storage** (บัญชีของคุณเอง)
- สถานะเช็คลิสต์ (ชื่อรายการ/ติ๊กแล้วหรือยัง) เก็บแยกไว้ในระบบจำค่าของหน้าเว็บนี้เอง

## ตั้งค่าครั้งแรก (ทำครั้งเดียว)

### 1. สร้าง Supabase Storage bucket

1. เข้า [supabase.com](https://supabase.com) → เปิดโปรเจกต์ (หรือสร้างใหม่)
2. ไปที่ **Storage** → **New bucket**
3. ตั้งชื่อ เช่น `accounting-docs` และเลือก **Private**

### 2. เปิดสิทธิ์ให้แอปอ่าน/เขียนไฟล์ใน bucket นี้

ไปที่ **SQL Editor** แล้วรัน (แก้ชื่อ bucket ให้ตรงกับที่ตั้งไว้):

```sql
create policy "app_access_accounting-docs"
on storage.objects for all to anon
using (bucket_id = 'accounting-docs')
with check (bucket_id = 'accounting-docs');
```

### 3. คัดลอกกุญแจเชื่อมต่อ

ไปที่ **Settings → API** แล้วคัดลอก:
- **Project URL** (เช่น `https://xxxxx.supabase.co`)
- **anon public key** — ใช้ตัวนี้เท่านั้น **ห้ามใช้ service_role key** เด็ดขาด เพราะ service_role มีสิทธิ์เต็มทั้งโปรเจกต์ ไม่ควรฝังในหน้าเว็บที่รันบนเบราว์เซอร์

### 4. กรอกในแอป

เปิดหน้าเว็บ (จากเครื่องหรือลิงก์ที่โฮสต์ไว้) → กดแถบ **"Supabase sync"** ด้านบน → กรอก Project URL, anon key, ชื่อ bucket → กด **"บันทึกและทดสอบเชื่อมต่อ"**

## วิธีใช้งานประจำเดือน

1. เลื่อนเดือนด้วยลูกศร ‹ › ให้ตรงกับเดือนที่จะส่ง
2. กด **แนบไฟล์** ในแต่ละรายการ (รองรับ PDF, JPG, PNG) — ไฟล์ภาพใหญ่กว่า 2MB และ PDF ใหญ่กว่า 10MB จะถูกบีบอัดให้อัตโนมัติ
3. ไฟล์จะอัปโหลดขึ้น Supabase ให้อัตโนมัติ (ขึ้นสถานะ "Supabase ✓")
4. เมื่อแนบครบ กด **📎 รวมไฟล์ทั้งหมดเป็น PDF**
5. กด **⬇ ดาวน์โหลด PDF รวม** หรือ **🖨 เปิดเพื่อปริ้นท์**

ปิดหน้าเว็บแล้วเปิดใหม่ (หรือเปิดจากอุปกรณ์อื่นที่กรอก Supabase เดียวกัน) ระบบจะดึงไฟล์ที่เคยแนบไว้กลับมาให้อัตโนมัติ ไม่ต้องอัปโหลดซ้ำ

## จะโฮสต์เป็นเว็บไซต์ผ่าน GitHub Pages ก็ได้ (ไม่บังคับ)

1. สร้าง repo ใหม่บน GitHub
2. อัปโหลดไฟล์นี้เข้า repo แล้วเปลี่ยนชื่อเป็น `index.html`
3. ไปที่ **Settings → Pages** → เลือก source เป็น branch `main`, โฟลเดอร์ `/ (root)` → Save
4. รอสักครู่จะได้ลิงก์ `https://ชื่อบัญชี.github.io/ชื่อ-repo/`

**ข้อควรรู้:** ถ้า repo เป็น public โค้ดหน้าเว็บจะเปิดดูได้ทุกคน แต่ anon key ไม่ได้ถูกฝังในไฟล์ (กรอกแยกทีหลัง เก็บในเบราว์เซอร์ผู้ใช้เอง) เอกสารจริงยังปลอดภัยตราบใดที่ตั้ง policy ของ bucket ไว้ถูกต้อง ถ้าต้องการเข้าถึงได้เฉพาะคุณคนเดียว แนะนำตั้ง repo เป็น private (ต้องมีแผน GitHub Pro ขึ้นไปถึงจะใช้ Pages แบบ private ได้) หรือเปิดไฟล์จากเครื่องโดยตรงก็ใช้งานได้ครบเหมือนกัน ไม่จำเป็นต้องพึ่งเว็บ

## ข้อจำกัดที่ควรรู้

- ขนาดไฟล์: ค่าเริ่มต้นของ Supabase มักจำกัดไฟล์ละ ~50MB (ปรับเพิ่มได้ที่ตั้งค่า bucket หรืออัปเกรดแพลน)
- การพรีวิวในแชท Claude อาจถูกจำกัดการเชื่อมต่อเน็ตเวิร์กออกนอกระบบ — แนะนำเปิดผ่านลิงก์ GitHub Pages หรือไฟล์ที่ดาวน์โหลดมาเปิดเองเพื่อความเสถียร
- token/anon key เก็บไว้ในเบราว์เซอร์ที่ใช้งานเท่านั้น หากเปลี่ยนอุปกรณ์ต้องกรอกใหม่
